    3. Continuous Monitoring (48 hours)¶

    Every 5 Minutes (Automated)¶

    Telemetry check: 

    from yunmin.core.dry_run_engine import DryRunEngine

engine = DryRunEngine()
telemetry = engine.get_telemetry()

# Alert if critical
if telemetry['daily_dd'] > 0.02:
    send_alert(level="CRIT", message=f"DD exceeded: {telemetry['daily_dd']:.2%}")

if telemetry['ws_stale_s'] > 60:
    send_alert(level="CRIT", message=f"WebSocket stale: {telemetry['ws_stale_s']}s")

if telemetry['rest_errors_1m'] >= 2:
    send_alert(level="CRIT", message=f"REST errors: {telemetry['rest_errors_1m']}")

    8. Emergency Procedures¶

    Market Crash (> 10% daily)¶

    Automatic: - Kill-switch triggers if DD > 2% - All positions closed

    Manual override (if needed): 

    python -c "
from yunmin.core.executor import Executor
executor = Executor()
executor.emergency_stop()
print('âœ… Emergency stop activated')
"

    Exchange Outage¶

    WebSocket disconnection: - Automatic reconnect (exponential backoff) - Alert after 5 failed attempts - Kill-switch if stale > 60s

    REST API error: - Automatic retry (3Ã—) - Alert on consecutive errors (â‰¥ 2) - Manual intervention after 10 errors

    Network Issues¶

    ISP down: - Kill-switch activates (no new orders) - Alert sent to team - Wait for network recovery - Manual restart once stable

    VPN issues: - Switch to direct connection - Or restart connection - Monitor latency before resuming
